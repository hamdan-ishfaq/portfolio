The Challenge

Traditional Neural Architecture Search (NAS) approaches are computationally prohibitive, often requiring thousands of GPU hours to discover optimal architectures for specific hardware constraints. This project aims to democratize NAS by introducing a distributed asynchronous framework.

Technical Approach

We utilize a controller based on a Recurrent Neural Network (RNN) combined with Proximal Policy Optimization (PPO) to sample candidate architectures. These candidates are dispatched to a cluster of worker nodes using a custom zeroMQ-based message passing interface.

  • Asynchronous Evaluator: Decouples candidate generation from evaluation.
  • Early Stopping Predictor: Uses a lightweight surrogate model to prune underperforming architectures within the first 5 epochs.
  • Fault Tolerance: Implemented checkpointing ensures cluster node failures do not halt the global search process.

def initialize_distributed_controller(config):
    # Setup ZMQ context
    context = zmq.Context()
    socket = context.socket(zmq.ROUTER)
    socket.bind(f"tcp://*:{config.port}")
    
    logger.info("Controller initialized. Waiting for workers...")
    return ControllerManager(socket, config.search_space)

Spent the day tuning the hyperparameters of the early-stopping surrogate. Found that a simple GCN (Graph Convolutional Network) over the architecture DAG outperforms the MLP baseline.

v0.8.3 2 weeks ago Fix

OOM Killer on Evaluation Nodes

Discovered a memory leak in the PyTorch graph cleanup during continuous evaluation loops. Enforced explicit `torch.cuda.empty_cache()` at the end of each worker evaluation cycle.
